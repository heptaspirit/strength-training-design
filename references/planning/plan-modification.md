# 现有计划修改参考

## 概述

用户在使用计划后，可能会提出修改需求。本章节提供修改现有计划的完整工作流。

---

## 触发场景

以下情况需要修改现有计划：

1. **用户主动提出意见**：
   - "太累了做不完" → 需要降低容量
   - "刺激不够" → 需要增加容量或强度
   - "有氧太累想换时间" → 需要调整有氧安排
   - "某个动作不舒服/疼痛" → 需要替换动作

2. **用户提供了已完成的一轮计划**：
   - 可作为参考，判断用户水平和训练习惯
   - 对比新旧计划，识别用户偏好

3. **用户在使用过程中遇到问题**：
   - 恢复不足（连续多日疲劳）
   - 时间冲突（某天突然没时间训练）
   - 设备限制（健身房某器械不可用）

---

## 修改工作流

### 第一步：收集修改需求

**礼貌询问用户具体需求**：

```
"我理解你想修改计划。为了更好地帮你调整，能具体说说：

1. 你觉得当前计划的哪个部分有问题？
   - 容量太大（做不完）？
   - 强度太高（RPE 经常超标）？
   - 某个动作不舒服或有疼痛？
   - 时间安排有问题（某天突然没时间）？

2. 你现在执行计划多久了？感觉恢复情况如何？

3. 是否有具体的训练日志记录（RPE、重量、次数）？如果有的话，可以发给我，我能更准确地帮你调整。"
```

**需求分类**：

| 需求类型 | 调整方向 | 参见 |
|---------|---------|------|
| **容量太大** | 减少组数或动作 | 第二节：容量调整 |
| **强度太高** | 降低 TS 重量或 RPE 目标 | 第三节：强度调整 |
| **刺激不够** | 增加容量或强度 | 第四节：容量/强度增加 |
| **动作不适** | 替换动作 | 第五节：动作替换 |
| **时间安排** | 调整训练日或内容 | 第六节：时间调整 |
| **有氧问题** | 调整有氧时间/类型 | 第七节：有氧调整 |

### 第二步：容量调整（降低）

**适用场景**：用户反馈"太累了做不完"、"每次训练都超时"

**调整原则**：
1. **优先减少辅助动作**，主项通常保留
2. **减少组数而非次数**（次数影响技术练习）
3. **遵循 JTS 退阶原则**（见 `SKILL.md` "退阶方案与自我调节"）

**调整方法**：

| 调整对象 | 原安排 | 调整后 | RPE 变化 |
|---------|---------|--------|----------|
| 辅助动作组数 | 3×8 | 2×8 | RPE 降低 0.5-1 |
| 辅助动作组数 | 3×8 | 3×6 | RPE 降低 0.5 |
| BO 组数 | 3 组 | 2 组 | RPE 降低 1 |
| BO 组数 | 2 组 | 1 组 | RPE 降低 1.5 |
| 移除某个辅助动作 | 3 个辅助动作 | 2 个辅助动作 | RPE 降低 1-2 |

**示例**：

```
用户反馈：W5 深蹲日太累，每次都超过 90 分钟，最后几个辅助动作都做不动。

调整方案：
- 原安排：深蹲主项 5×8 + 暂停深蹲 3×8 + 腿举 3×10 + 腿伸展 3×12
- 调整后：深蹲主项 5×8 + 暂停深蹲 2×8 + 腿举 2×10
- 移除：腿伸展（容量降低约 30%）
- 预期：训练时长降至 70-80 分钟，RPE 降低 1-1.5
```

### 第三步：强度调整（降低）

**适用场景**：用户反馈"RPE 经常超标"、"TS 组经常比预期高 2 点以上"

**调整原则**：
1. **优先降低 BO 重量**，TS 重量通常保留（探测功能）
2. **降低 RPE 目标**（如从 RPE 8.5 降至 RPE 8）
3. **插入额外减载周**（如果连续 2 周 RPE 超标）

**调整方法**：

| 调整对象 | 原安排 | 调整后 | 效果 |
|---------|---------|--------|------|
| TS 重量 | 117.5×5 @ RPE 8.5 | 115×5 @ RPE 8 | 降低 RPE 0.5-1 |
| BO 重量 | TS 重量 × 88% | TS 重量 × 85% | BO RPE 降低 1-2 |
| RPE 目标 | 8.5 | 8 | 整体容量降低 |
| 插入减载周 | 无 | W5 后插入 1 周减载 | 恢复疲劳 |

**示例**：

```
用户反馈：W5 卧推 TS 117.5×3 @ RPE 9.5（预期 RPE 8.5），BO 也跟着很累。

调整方案：
- TS 重量：117.5kg → 115kg（降低 2.5kg）
- BO 重量：105kg → 100kg（降低 5kg，从 TS 88% → 87%）
- RPE 目标：8.5 → 8（降低 0.5）
- 预期：TS RPE 降至 8.5-9，BO RPE 降至 7-8
```

### 第四步：容量/强度增加（刺激不够）

**适用场景**：用户反馈"刺激不够"、"某个肌群感觉不到发力"、"进步停滞"

**调整原则**：
1. **优先增加辅助动作组数**，主项重量通常已按周期递增
2. **增加次数而非组数**（次数增加 → 代谢压力增加 → 肌肥大）
3. **检查 MRV**，确保不超过 MRV

**调整方法**：

| 调整对象 | 原安排 | 调整后 | 效果 |
|---------|---------|--------|------|
| 辅助动作组数 | 2×8 | 3×8 | 容量增加 50% |
| 辅助动作次数 | 3×8 | 3×10 | 容量增加 25% |
| 增加辅助动作 | 2 个辅助动作 | 3 个辅助动作 | 容量增加 30-50% |
| 缩短组间休息 | 3 分钟 | 2 分钟 | 密度增加，代谢压力增加 |

**示例**：

```
用户反馈：W5 卧推后胸部感觉不够（刺激不够），但三头肌感觉很累。

分析：可能卧推过多用到三头肌，胸部参与不足。

调整方案：
- 增加胸部辅助动作：加入夹胸 3×12 @ RPE 7
- 或减少三头肌参与：卧推握距略增宽（2.5-5cm）
- 预期：胸部感觉增强，卧推重量可能略降但更稳定
```

### 第五步：动作替换

**适用场景**：用户反馈"某个动作不舒服"、"某个动作有疼痛"、"健身房没有某个设备"

**替换原则**：
1. **Same 肌群，不同动作**（如腿举 → 腿伸展）
2. **Same 动作模式，不同设备**（如杠铃深蹲 → 六角杠深蹲）
3. **避免疼痛动作**，优先康复/预防性动作
4. **疼痛 ≠ 完全停训**：参考 `pain-management.md`（BBM 主动康复策略），优先在疼痛耐受范围内修改训练而非直接排除动作

**常见替换表**：

| 原动作 | 替换动作 | 原因 |
|---------|---------|------|
| 深蹲（杠铃） | 六角杠深蹲 / 安全杠深蹲 | 腰背疼痛 |
| 卧推（杠铃） | 哑铃卧推 / 机器卧推 | 肩袖疼痛 |
| 硬拉（传统） | 六角杠硬拉 / RDL | 下背疼痛 |
| 高翻 | Clean Pull / 高抓 | 技术难度高/手腕疼痛 |
| 腿举 | 腿伸展 / 保加利亚分腿蹲 | 下背不适 |
| 农夫行走 | 握力器 / 死虫式（核心） | 场地限制 |

**示例**：

```
用户反馈：深蹲时腰背疼痛，可能是柔性组织问题。

调整方案：
- 替换动作：杠铃深蹲 → 六角杠深蹲（减少腰背参与）
- 或：加入更多腿举、腿伸展（减少深蹲组数）
- 建议：咨询物理治疗师，排除伤病
```

### 第六步：时间调整

**适用场景**：用户反馈"某天突然没时间训练"、"训练日冲突"

**调整原则**：
1. **重新分配训练日**，保持训练频率
2. **调整训练内容**，将时间冲突日的训练拆分或合并
3. **优先保证主项**，辅助动作可调整

**调整方法**：

| 原安排 | 冲突 | 调整后 |
|---------|------|--------|
| 周一/周三/周五 | 周三突然没时间 | 周一/周二/周五（调整一天） |
| 4 天/周 | 某天只能练 45 分钟 | 该天只练主项 + 1 个辅助动作 |
| 深蹲 + 硬拉同天 | 太累 | 拆分到两天 |

**示例**：

```
用户反馈：周三突然没时间训练（工作加班），但其他天可以。

调整方案：
- 原安排：周一深蹲、周三卧推、周五硬拉
- 调整后：周一深蹲、周二卧推、周五硬拉（周三内容移到周二）
- 注意：周一和周二连续训练，需监控疲劳
```

### 第七步：有氧调整

**适用场景**：用户反馈"有氧太累"、"没时间做有氧"、"有氧后影响力量训练"

**调整原则**：
1. **调整有氧时间**（见 `references/ohp-core-aerobic.md` "有氧训练时机"）
2. **更换有氧类型**（低冲击 vs. 高冲击）
3. **调整有氧时长**或**移除有氧**（如用户恢复不足）

**调整方法**：

| 用户反馈 | 调整方案 |
|---------|---------|
| "练腿日后再做有氧太累" | 有氧改到非训练日，或练腿日只做 10 分钟 LISS |
| "没时间做有氧" | 有氧时长从 30 分钟降至 15-20 分钟，或增加 NEAT（日常活动） |
| "有氧后影响力量训练" | 有氧与力量训练隔开至少 1 天，或改为力量训练后 10 分钟 LISS |
| "跑步膝盖疼" | 更换有氧类型：跑步 → 自行车/游泳（低冲击） |

**示例**：

```
用户反馈：深蹲日后再做 30 分钟自行车太累，每次都超时。

调整方案：
- 原安排：深蹲日训练后 30 分钟 LISS
- 调整后：深蹲日训练后 10 分钟 LISS（仅促进恢复），30 分钟 LISS 改到非训练日（如周二/周四）
- 预期：深蹲日时长降至 90 分钟以内，恢复改善
```

## 参考以往计划

### 为什么参考以往计划？

**价值**：
1. **判断用户水平**：以往计划的容量/强度反映用户恢复能力
2. **识别训练习惯**：用户偏好哪些动作、哪些时间安排
3. **避免重复错误**：以往计划的问题（如过度疲劳）不再犯

### 如何参考？

**第一步：询问用户是否有以往计划**

```
"你是否之前完成过一轮训练计划？如果有的话，可以发给我看看，我能更准确地判断你的水平和训练习惯，设计出更适合你的新计划。"
```

**第二步：分析以往计划**

| 分析维度 | 关注点 | 如何调整新计划 |
|---------|-------|----------------|
| **容量** | 以往计划每周总组数 | 如果用户以前能承受高容量（接近 MRV），新计划可适当增加容量；如果用户以前容量低（<60% MRV），新计划从低容量开始 |
| **强度** | 以往计划 TS 重量/RPE | 如果用户以前 TS RPE 经常超标，新计划降低强度或增加减载频率 |
| **动作选择** | 用户偏好哪些辅助动作 | 新计划保留用户喜欢的动作，替换用户不喜欢的动作 |
| **时间安排** | 用户以前每周训练几天、哪些天 | 新计划遵循用户习惯的时间安排 |
| **恢复情况** | 用户以前是否经常疲劳、是否需要额外减载 | 如果用户恢复差，新计划增加减载频率或降低 MRV 预期 |

**第三步：结合用户反馈调整**

- 如果以往计划"太累" → 新计划降低容量 10-20%
- 如果以往计划"刺激不够" → 新计划增加容量 10-20%
- 如果以往计划"某个动作不舒服" → 新计划替换该动作

**示例**：

```
用户提供了以往 8 周计划：
- 容量期：5×8 深蹲 + 3×8 暂停深蹲 + 3×10 腿举（每周 22 组股四）
- 用户反馈：W2 后就开始疲劳，W3 经常做不完

分析：用户股四 MRV 可能低于 20 组/周（基准），需降低容量。

新计划调整：
- 容量期：5×8 深蹲 + 2×8 暂停深蹲（移除腿举）
- 每周股四容量：15 组（降低 32%）
- 预期：用户能完成，RPE 降低 1-2
```

## 修改计划检查清单

修改计划后，务必检查：

1. ✅ **修改需求明确**：用户具体反馈了什么？需要调整什么？
2. ✅ **调整符合 JTS 原则**：是否遵循 JTS 退阶原则？是否优先减少辅助动作？
3. ✅ **MRV 审计**：调整后各肌群容量是否仍然 ≤ MRV（含个性化调整）？
4. ✅ **RPE 预期**：调整后 RPE 是否符合预期（容量期 RPE 6-7，力量期 RPE 7.5-8.5）？
5. ✅ **时间安排合理**：调整后训练时长是否控制在用户要求范围内？
6. ✅ **动作替换合理**：替换动作是否针对同一肌群？是否避免疼痛动作？

---

## 常见错误

1. ❌ 用户反馈"太累"，直接降低主项重量 → ✅ 优先减少辅助动作，主项重量通常保留
2. ❌ 用户反馈"刺激不够"，直接增加主项组数 → ✅ 优先增加辅助动作组数或次数
3. ❌ 替换动作时，选择完全不同肌群的动作 → ✅ 同一肌群，不同动作/设备
4. ❌ 调整有氧时，不考虑对力量训练的影响 → ✅ 有氧与力量训练隔开至少 1 天
5. ❌ 参考以往计划时，不结合用户反馈 → ✅ 以往计划 + 用户反馈 = 最准确的调整依据
